Dynamics 365 popup sign-in loop · reproduction

This page reproduces how MSAL.js, after silent token acquisition fails (AADSTS700084 / AADSTS160021), falls back to an interactive window.open popup. It uses real browser channels to pass the authorization code from the popup back to this tab; whichever channel is severed shows up directly in the "Callback channels" cards and the console below. In a normal browser the callback succeeds and the loop ends; in a browser that isolates tabs, if the callback can't get through, the "unable to open window" warning appears and re-triggers on every Close click.
