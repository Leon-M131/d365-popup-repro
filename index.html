Dynamics 365 popup sign-in loop · reproduction

This harness reproduces the MSAL popup flow. On Sign in it simulates the silent-token failures (AADSTS700084, then AADSTS160021), then opens a real window.open popup. The popup makes a genuine cross-origin round trip — app origin → a different "identity provider" origin (htmlpreview.github.io) → back to the app origin's redirect.html#code=… — exactly like app → login.microsoftonline.com → redirect_uri. The parent then tries to read the code back over four channels; whichever is severed shows up in the cards and console.
